# 請求書管理アプリ 要件定義書

## プロダクト概要

広告・Web制作業向けのシンプルなクラウド請求書管理アプリ。
board の業務設計と Misoca のシンプルな UX を参考に、ミニマムな機能から開始して拡充していく。

---

## スコープ（MVP）

### 対象ユーザー
- 広告・Web制作の個人・小規模チーム（1〜5名）
- 顧客への請求書発行・管理が主目的

### スコープ外（MVP）
- 売上分析・発注管理・発注先管理
- 捺印申請ワークフロー
- 複数ユーザー権限管理
- 会計ソフト連携

---

## 機能要件

### 1. 顧客管理

| 項目 | 仕様 |
|------|------|
| 顧客一覧 | 顧客名・住所・TEL・送付方法・アーカイブ表示 |
| 顧客登録・編集 | 下記フィールド参照 |
| 郵便番号補完 | 郵便番号入力で住所を自動補完（zipcloud API） |
| アーカイブ | 削除せず非表示化 |

**顧客フィールド**
- 顧客名（書類表示用・必須）
- 顧客略称名（画面表示用・必須）
- 敬称（御中 / 様）
- 郵便番号・住所
- TEL / FAX
- メールアドレス（書類送付先）
- 担当者名・部署名
- デフォルト支払条件
- デフォルト振込口座
- 書類送付方法（メール / 郵送）
- インボイス登録番号
- 備考

---

### 2. 案件管理

| 項目 | 仕様 |
|------|------|
| 案件一覧 | 案件名・顧客・見積額・請求額・受注ステータス・進捗・請求日・アクション |
| 案件登録・編集 | 下記フィールド参照 |
| 受注ステータス | 見積中（高/中/低）/ 受注確定 / 受注済 / 失注 |
| 行背景色 | ステータスに応じた自動色分け |
| 行末ショートカット | 「書類」「編集」ボタン |
| 登録後の導線 | 「登録して書類へ」ボタンで書類編集に直遷移 |

**案件フィールド**
- 案件名（必須）
- 顧客（必須）
- 担当者名（顧客から自動セット）
- 受注ステータス（ラジオボタン）
- 請求タイミング（一括 / 分割）
- 請求（予定）日（「今月末」「来月末」ショートカットボタン）
- 支払期限
- 支払条件（顧客から自動セット・編集可）
- 振込口座（顧客から自動セット・編集可）
- 案件区分（任意タグ）
- 社内メモ

---

### 3. 書類作成

#### 作成できる書類（MVP）
- 見積書
- 請求書
- 領収書

#### 明細構造（重要）
- **行タイプ**：各明細行は以下の3種類から選択
  - `通常`：摘要・数量・単価・税率・金額（=数量×単価）を持つ標準行
  - `見出し行`：摘要のみ表示。グループの大分類見出し（例：企画構成 / デザイン）として使用
  - `小計行`：直前の見出し行（または書類冒頭）から自身までの `通常` 行の金額を合算して表示。値は永続化せず保存時に再計算
- グループ概念は別エンティティを持たず、`見出し行 → 通常行 × N → 小計行` のシーケンスで表現
- 明細行のドラッグ並び替え
- 各行の右端に `⋮` メニュー（並び替え／直下に行追加／複製／削除）
- テーブル上部に `行を追加` `行を複製` のクイックアクションボタン

**明細行フィールド**
- 行タイプ（通常 / 見出し行 / 小計行）
- 摘要
- 数量・単価（通常行のみ）
- 消費税率：10% / 軽減8% / 非課税（通常行のみ）
- 金額（自動計算）
- 備考

**書類ヘッダー**
- 書類番号（自動採番）
- 発行日
- 支払期限 / 有効期限（書類タイプにより使い分け）
- 件名（案件名から自動セット）
- 宛名（顧客から自動セット・スナップショット保持）
- 振込先
- 顧客向け備考（書類PDFに出力される）
- **社内メモ**（書類PDFには出力しない・自分用）
- テンプレート選択（書類デザインの切替・MVPは1種）
- 小計・消費税・合計（自動計算）
- 源泉徴収（ON/OFF）
- インボイス対応（登録番号・税率別内訳表示）

#### PDF出力
- ブラウザ印刷 or サーバーサイド PDF 生成
- デザインテンプレート：シンプルな1種類（MVP）

---

### 4. 請求管理

| 請求ステータス | 遷移トリガー |
|--------------|-------------|
| 下書き (draft) | 初期状態。書類編集中・確定前 |
| 発行済 (issued) | 「見積書を発行 / 請求書を発行」ボタンで確定。PDF/メール送付可能になる |
| 送付済 (sent) | メール送付と同時に自動変更 / 手動変更可 |
| 入金済 (paid) | 手動で入金確認操作（請求書のみ） |

**機能**
- 編集画面のヘッダーアクション：`プレビュー` / `下書き保存` / `[書類タイプ]を発行 ▼`
- 請求一覧（請求月・ステータスでフィルタ）
- 未請求アラート（ダッシュボードに件数表示）
- メール送付（顧客のメールアドレスに自動セット・本文テンプレート）
- 入金確認日の記録

---

## 非機能要件

| 項目 | 仕様 |
|------|------|
| 認証 | メール＋パスワード / Google OAuth |
| レスポンシブ | PC優先（モバイルは参照のみ可） |
| データ保持 | 書類作成時の顧客情報をスナップショット保存 |
| セキュリティ | HTTPS・認証必須 |
